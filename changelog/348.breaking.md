The `dag` argument of `TabPFNUnsupervisedModel.generate_synthetic_data()` and `impute()` must now specify every feature as a key; parentless features are declared explicitly with an empty list (`{i: []}`). Partial DAGs previously gave unlisted features an empty parent list silently, sampling them marginally (independent of everything else) — rarely what a caller constraining a subset of features intends. An incomplete DAG now raises `ValueError` naming the missing features.
