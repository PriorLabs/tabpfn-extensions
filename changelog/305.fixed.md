`TabPFNUnsupervisedModel` now honors the classifier's true class limit (160 on TabPFN v3, read from `get_inference_config().MAX_NUMBER_OF_CLASSES` at construction, falling back to 10 for models that don't expose it) for categorical features end-to-end — both when retaining declared categorical columns during `fit` and when routing them to the classifier vs. regressor. Previously a dead `max_num_classes_` lookup and a hard-coded threshold in `infer_categorical_features` silently capped this at 10, so categorical features with 10+ distinct values were treated as numerical.
