`TabPFNUnsupervisedModel.density_` now models the requested feature for the unconditional term of the chain-rule decomposition instead of always using the first column. Previously every feature permutation in `outliers` (and `outliers_pdf`/`outliers_pmf`) opened with the first column's marginal — double-counting feature 0 and dropping the first permuted feature's marginal — and `generate_synthetic_data` sampled the first generated feature of each permutation from the first column's distribution.
