Fixed `ClientTabPFNRegressor.predict(..., output_type="full")` raising on `tabpfn>=8.1`: the wrapper imported the removed `tabpfn.model.bar_distribution` module before calling the client, discarding the compatible `criterion` current tabpfn-client already returns. The client output's criterion is now passed through (and only reconstructed from `borders` for older clients), un-breaking e.g. `TabPFNUnsupervisedModel.outliers()` on the client backend. The wrapper also now forwards other output types such as `output_type="quantiles"` to the client instead of silently returning means.
